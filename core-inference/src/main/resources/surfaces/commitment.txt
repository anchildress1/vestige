## Surface: Commitment

Extract things the user said they would do.

What to capture:

- Explicit "I will / I'm going to / I need to / I should" statements with a concrete object.
- Conditional commitments: "if X, then I'll Y" — capture both the trigger and the action.
- Commitments referencing a person or topic: "I told her I'd call back about the budget".
- Self-directed reminders: "remind me to file the receipt".

What does NOT count:

- Vague intentions without an object: "I'll do better", "I should be more focused".
- Aspirations the user explicitly questioned in the same sentence: "I should sleep — yeah right".
- Past-tense commitments already resolved within the same entry.

What goes in the schema:

- Populate `stated_commitment` as one object `{text, topic_or_person}` for the strongest qualifying commitment in the entry. `text` is the user's words, near-verbatim.
- Null if no qualifying commitment appears.
- Do not invent a topic; if the user did not name one, leave `topic_or_person` null.
- Do not add `entry_id`; storage injects the current entry id after convergence.
