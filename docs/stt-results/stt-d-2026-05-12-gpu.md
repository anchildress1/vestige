## 🔴 GPU regressions vs CPU same-day run

### Reproducible parse failures — C2 + D1 INFERENTIAL

Same two entries failed on the GPU run from 2026-05-10. Two runs, two days apart, identical
failure pattern:

**C2** ("Task app decision still looping. I keep changing the weights and pretending that is progress.")
- LITERAL: parsed cleanly on attempt 1 → `[task-app, decision-loop]`
- INFERENTIAL: attempt 1 parse-fail → retry → attempt 2 parse-fail → **gave up**
- SKEPTICAL: attempt 1 parse-fail → retry → attempt 2 parse-fail → **gave up**

**D1** ("3:12am. I am rearranging the notes app again instead of sleeping. This is not a system, it's a small administrative haunting.")
- LITERAL: parsed cleanly on attempt 1 → `[late-night, notes-app, rearranging]`
- INFERENTIAL: attempt 1 parse-fail → retry → attempt 2 parse-fail → **gave up**
- SKEPTICAL: parsed cleanly on attempt 1 → `[late-night, notes-app, rearranging]`

Common structural property in both texts: a reflective second clause ("I keep changing...",
"This is not a system..."). Hypothesis: the inferential lens's prompt structure interacts
badly with that text shape on GPU sampling but not on CPU.

### Soft regressions — A6, C3, D3, D1

Four CPU-meaningful entries collapsed to GPU-not-meaningful. Three of them (A6, C3, D3) are
"soft" — the parse succeeded but all three lenses converged on identical output. The fourth (D1)
is a hard regression caused by INFERENTIAL parse-fail.

Possible causes for soft regressions (untested):

1. **GPU FP16 vs CPU FP32 precision** — narrow-margin token-distribution differences may make
   the GPU more deterministic across sampled lens calls.
2. **Lens-prompt sensitivity** — same prompt assembly path on both backends, but the model's
   responsiveness to the persona-system-prompt steering may be precision-dependent.
3. **Greedy convergence under GPU sampling** — GPU paths could converge faster on the highest-
   logit tag set, hiding the LITERAL-vs-INFERENTIAL split that CPU surfaces.

These are hypotheses. Verifying any would require running each lens 3× and measuring per-lens
consistency on both backends — out of scope for this run.
